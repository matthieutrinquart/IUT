    TP 01&02 - Table de hachage et complexité (version 2019)

    Les tables de hachage

    L’objectif de ce TP est d’écrire un programme qui permet de calculer le nombre d’occurence des mots présents dans un fichier texte. Nous utiliserons ici le texte de Jules Verne “Voyage au centre de la terre”.

    Ensuite, le programme devra permettre de saisir un mot et d’afficher son nombre d’occurence.

    Pour la partie calcul du nombre d’insertion, un algorithme naïf consisterait à comparer tous les mots deux à deux, ce qui conduirait à une complexité en O(n2).

    Au cours de ce TP nous allons utiliser le conteneur unordered_map disponible dans la STL pour compter le nombre d’occurrence de chaque mot. Ce conteneur implante en fait une table de hachage. En théorie, nous devrions donc avoir une compléxité de l’ordre de O(1) si tout va bien, O(n) dans le pire des cas. Pour vérifier cette hypothèse vous utiliserez les unordered_map pour stocker des pairs <clé,valeur> où la clé sera le mot et la valeur son nombre d’occurrence dans le texte.

    Vous allez charger le texte de Jules Verne disponible avec le sujet du TP en le lisant mot par mot. Vous pouvez utiliser pour ça un ifstream ifs; et l’opérateur d’injection de flux ifs>>std::string. Vous pouvez également utiliser la fonction ifs.good() qui est bien utile dans ce genre de situation. Pour chaque mot que vous trouverez, vous regarderez s’il est présent dans le conteneur et si tel est le cas, vous augmenterez sa valeur d’occurrence de 1, sinon, vous l’ajouterez avec comme valeur d’occurrence 1.

    Ensuite, vous compléterez votre programme afin que l’on puisse rechercher un mot et afficher son nombre d’occurence.

    Une fois cette première partie terminée, vous vous intéresserez à la compléxité de votre algorithme. L’objectif ici est de réaliser une série de mesures pour valider la base théorique. Pour cela, vous devez mesurer le temps d’exécution de votre programme, tant pour la partie decompte du nombre d’occurence qui correspond à l’insertion dans notre cas, que pour la partie recherche d’un mot. Pensez à faire varier la nature et le nombre de motslors de vos mesures, sinon cela n’a pas de sens. La classe chrono::steady_clock disponible depuis la norme 2011 vous sera bien utile pour obtenir des mesures de l’ordre de la nanoseconde (Cf. std::chrono::nanoseconds).

    Une fois les mesures effectuées, vous pouvez les rediriger dans un fichier texte, soit depuis la sortie standard en utilisant l’opérateur de redirection de flux > lors de l’appel à l’exécutable, soit en utilisant un ofstream. Pour obtenir un graphique à partir de ces données, utilisez un tableau ou un outil en ligne de commande tel que gnuplot qui feront très bien l’affaire.

    Bonus
    Une fois le travail terminé avec une unordered_map, vous pouvez réaliser la même chose avec une map afin de comparer les temps en insertion et en recherche.

    Remarque
    Si vous travaillez avec la console de Windows, comme le fichier en entrée est encodé en ANSI (Windows-1252), vous devez modifier le code page utilisé pour que les caractères spéciaux soient affichés correctement (é,à,è,ç, etc.). Pour cela vous devez utiliser la commande chcp. Par défaut vous devriez être en Latin 1 (code page 850). Le code page correspondant à l’encodage Windows-1252 est le code page 1252. En tapant la commande chcp 1252 dans la console, celle-ci utilisera désormais l’encodage Windows-1252.
